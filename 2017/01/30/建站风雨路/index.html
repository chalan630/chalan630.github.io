建站风雨路

发表于 2017-01-30
字数统计: 1,190 | 阅读时长 ≈ 5

建站的准备

  1. 建一个自己的网站需要云服务器和一个自己的域名。
  2. LAMP/LNMP/WAMP/WNMP的选择
  • 注:Mac不包括在内
  1. WIX/WordPress/discuz
  • 另：建站模版不局限于上述三种，如HEXO+Github、Jekyll、等等。

建站的实操

域名

在腾讯云购买的服务器和域名。

  • PS:学生机有优惠券

域名的解析

  • 域名一定要有解析记录

    所谓解析记录简单的来说就是将你的域名和你的主机公网ip联系起来，访问你的域名时自动转到访问你的主机

进行服务器的配置

创建云主机

选择镜像Centos6.5 (因为想试试动手能力就没有选择建站模版)

登录设置选择SSH密钥，这样可以用一些软件通过SSH远程登录你的服务器，例如：Windows上的Xshell,Android上的JuiceSSH等等。

进行服务器环境的配置

  1. LAMP环境设置 安装Apache、Mysql、PHP。

    yum install httpd mysql mysql-server php php-mysql php-gd php-xml

    CentOS上可以使用命令yum install进行安装，-y 表示安装过程中全部选择yes

  2. 设置httpd、mysqld为开机启动服务

    chkconfig httpd on
chkconfig mysqld on

  3. 启动服务

    service httpd start
service mysqld start

  4. MySQL数据库设置
    （1）设置

    mysql_secure_installation

    主要内容如下
    1 Set root password? [Y/n] //设置root密码（最好自己设置密码，选Y）
    2 anonymous users? [Y/n] //删除匿名用户（选Y）
    3 Disallow root login remotely? [Y/n] //禁止root用户远程登录（选n）
    4 Remove test database and access to it? [Y/n] //删除默认的 test 数据库（选Y）
    5 Reload privilege tables now? [Y/n] //是否马上应用最新的设置（选Y）

    （2）登录

    mysql –u root –p   （一开始密码为空，直接按回车键即可）      
mysql> create database wordpress;   //创建wordpress数据库，为下面安装wordpress做准备

    ####5. 安装wordpress
    （1）下载安装

    wget http://cn.wordpress.org/wordpress-4.7.2-zh_CN.zip //下载
    unzip wordpress-4.7.2-zh_CN.zip //解压
    cp -rf wordpress/* /var/www/html/ //将wordprss下所有的文件复制到apache服务器下的根目录

（2）配置wordpress的配置文件

cd /var/www/html/wordpress/             //进入wordpress文件夹下
cp wp-config-sample.php wp-config.php   //复制配置文件
vim wp-config.php                       //编辑wordpress的配置文件

然后输入数据库名称，例如我上面创建的数据库wordpress，然后是数据库的用户名和密码，“MySQL主机”一般默认为localhost，不需要修改

我遇到的问题

Wordpress不能自动更新、安装主题、插件。

wordpress遇到以下问题时：
1、不能上传图片
2、不能自动安装主题、插件（需要FTP账户）
3、不能自动更新
4、其它任何需要wordpress写文件的问题

这些问题基本都是一个原因，你的wordpress目录不属于当前的用户和组，即web访问的用户没有权限操作wp的一切需要写权限的操作，其实就是linux下权限不足，无法写入造成的。

解决方式就是更改文件所有者

cd /var/www/html/

给予所有的写权限：

chmod 777 wp-content

接下来给你的博客的文章上传一张图片，WP会生成一个目录，然后查看是哪个用户创建了文件夹。一般情况下，这个用户名叫apache，也有不少人发现这个用户是nobody
进入到wp的wp-content目录，查看该目录下所有文件/文件夹的权限，所属用户、用户组：

cd wp-content
ls -l

由于上图时我已经解决问题，原本上传目录uploads是用户apache创建的，其他目录为root用户创建

接下来把wp-content权限还原到 755

cd ..
chmod 755 wp-content

下来就是实际修复的命令了，改变wp所在文件夹的拥有者为刚找到的这个用户apache：

cd ..
chown -R apache:apache wp-content

修改固定链接后无法打开网页的问题

使用wordpress搭建网站，为了让文章URL看起来漂亮一点，wordpress提供了设置->固定链接设置->选择你喜欢的url结构。所以，我们一般为了用户看着这个url顺眼一点，会进行url地址的自定义。

但是那就是要让Apache支持mod_rewrite模块，而默认情况下Apache的相关配置配置并没有开启。

解决方案：开启Mod_rewrite模块方法：

  1. 直接打开httpd.conf

    httpd -V （查看apache安装位置）

    在我的Centos中，Apache配置文件在/etc/httpd/conf/httpd.conf

  2. 搜索 LoadModule rewrite_module modules/mod_rewrite.so 去掉前面的#

    在vim中搜索用/

  3. 查找AllowOverride属性

    /AllowOverride

    修改所有AllowOverride属性的值为ALL

  4. 重启apache服务

    service httpd restart
